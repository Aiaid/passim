# Epic 8: 共享用户管理

> Phase: 5 (P3)

Passim 是单管理员系统——只有一个人（API Key 持有者）管理一切。但通过应用分享功能（US-3.5），管理员可以把 VPN、网盘等应用的访问权分享给朋友。Epic 8 提供一个集中的地方来管理"我分享给了谁、分享了什么"。

**和 US-3.5/US-3.6 的关系：**
- US-3.5 分享给朋友 — 在单个应用内发起分享
- US-3.6 管理已分享 — 在单个应用内管理分享链接
- **US-8.1** — 跨所有应用的分享总览，"我的朋友们都在用什么"
- **US-8.2** — 全局统计，资源占用概览

---

## US-8.1 共享用户总览 `P3` `Phase 5`

**作为** 管理员
**我想** 在一个地方看到我分享给所有朋友的访问情况
**以便** 统一管理，不用一个一个应用去看

**验收标准:**

**按人聚合的视图:**
- [ ] Settings → 共享管理 页面
- [ ] 按"人"聚合显示（同一个名字/标识的分享归为一组）:
  - 朋友名称/备注（分享时填的）
  - 正在使用的应用列表（如 "WireGuard, WebDAV"）
  - 每个应用的状态（🟢 活跃 / ⚪ 未使用 / 🔴 已过期）
  - 分享时间
- [ ] 点击某个人 → 展开显示该人的所有分享详情:
  - 每个应用的分享链接、到期时间、使用情况
  - 可单独撤销某个应用的访问
- [ ] [全部撤销] — 一键撤销某个人的所有访问

**快捷操作:**
- [ ] 直接从总览页给某人新增分享（选应用 → 生成链接）
- [ ] 批量撤销已过期的分享链接

## US-8.2 全局概览 `P3` `Phase 5`

**作为** 管理员
**我想** 看到整体的资源使用情况
**以便** 知道分享出去的东西占了多少资源

**验收标准:**

**统计卡片:**
- [ ] 共享用户数（有多少朋友在用）
- [ ] 活跃分享数 / 已过期
- [ ] 总节点数 / 在线节点数
- [ ] 总应用部署数

**资源占用（按应用）:**
- [ ] 每个应用的共享用户数和资源占用:
  - WireGuard: 3 个用户，带宽 XX
  - WebDAV: 1 个用户，存储 XX GB
- [ ] 帮助管理员判断"是不是分享太多了，资源扛不住"

**前端交互:**
```
Settings → 共享管理 → 概览

┌────────────┐ ┌────────────┐ ┌────────────┐ ┌────────────┐
│ 3 个朋友   │ │ 5 个分享   │ │ 2 个节点   │ │ 4 个应用   │
│ 在使用     │ │ 3 活跃     │ │ 全部在线   │ │ 已部署     │
└────────────┘ └────────────┘ └────────────┘ └────────────┘

应用资源占用
┌──────────────────────────────────────────────────────┐
│ 🛡 WireGuard      3 个用户 · 带宽占用低             │
│ 📁 WebDAV         1 个用户 · 已用 12 GB             │
```
