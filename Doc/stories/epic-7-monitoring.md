# Epic 7: 监控与可观测性

> Phase: 2 (P1) + Phase 4 (P2)

监控功能帮助用户回答三个问题："网速快不快？""HTTPS 正不正常？""域名能不能访问？"。大部分时候这些是自动运行的，用户不需要主动关注，但需要的时候能找到。

**设计原则：**
- 技术指标配上直觉化解读（856 Mbps → "非常快"，3.2 ms → "延迟极低"）
- SSL 和 DNS 状态用颜色和图标表达，正常时不打扰，异常时主动提醒
- 测速结果和证书状态放在节点详情页，不是独立页面
- 多节点总览已在 Dashboard (US-1.3) 中覆盖，本 Epic 专注于诊断工具

---

## US-7.1 节点监控仪表盘 `P1` `Phase 2`

**作为** 管理多台 VPS 的用户
**我想** 在一个页面总览所有节点的健康状态
**以便** 快速发现异常

> 注：单节点用户的 Dashboard 由 US-1.3 覆盖。本 story 是 US-1.3 在多节点场景下的扩展。

**验收标准:**

**节点卡片网格:**
- [ ] Dashboard 显示所有节点（本地 + 远程）的卡片网格
- [ ] 每张卡片显示:
  - 节点名称
  - IP 地址 + 国旗 emoji（根据 IP 地理位置）
  - 状态指示灯（🟢 在线 / 🔴 离线）
  - 在线时: CPU / 内存使用率进度条、已部署应用数
  - 离线时: "上次在线: 3 小时前"，灰色卡片样式
- [ ] 离线节点排在最前面（异常优先）
- [ ] 点击卡片 → 进入该节点的详情页

**顶部总览统计:**
- [ ] 总节点数 / 在线 / 离线
- [ ] 总应用数 / 运行中 / 已停止
- [ ] 有节点离线时，统计栏显示黄色警告背景

**单节点时:**
- [ ] 只有一个本地节点时，不显示卡片网格，直接显示 US-1.3 的 Dashboard 视图
- [ ] 添加第一个远程节点后自动切换为多节点视图

## US-7.2 测速 `P2` `Phase 4`

**作为** 用户
**我想** 测试 VPS 的网络速度
**以便** 知道网速够不够快

测速分三种场景，解决不同问题：
1. **你 → 本地节点** — "我到这台机器的网速快不快？"
2. **你 → 远程节点** — "我到远程那台机器的网速快不快？"（浏览器/App 直连远程节点测速）
3. **节点 ↔ 节点** — "两台机器之间的内网速度怎么样？"（用于评估跨节点部署的性能）

**验收标准:**

**① 客户端 → 当前节点测速:**
- [ ] 节点详情页 → "网络测速" 区域
- [ ] [开始测速] 按钮
- [ ] 测速进行中显示实时进度动画（下载 → 上传 → 延迟，依次进行）
- [ ] 测速结果显示:
  - 下载速度 + 直觉评价（如 "856 Mbps · 非常快"）
  - 上传速度 + 直觉评价（如 "421 Mbps · 很快"）
  - 延迟 (带 tooltip: "数据从你到服务器的往返时间，越低越好")
  - 抖动 (带 tooltip: "延迟的波动幅度，越低越稳定")
- [ ] 速度评价标准:
  - 非常快: > 500 Mbps
  - 很快: 100-500 Mbps
  - 一般: 10-100 Mbps
  - 较慢: < 10 Mbps
- [ ] 上次测速时间
- [ ] 测速历史（最近 10 次，简单列表）

**② 客户端 → 远程节点测速（有远程节点时）:**
- [ ] 远程节点详情页同样有 "网络测速" 区域和 [开始测速] 按钮
- [ ] 浏览器/App 直接对远程节点的测速端点发起请求（不经过本地节点中转）
- [ ] 结果格式和①一致（下载/上传/延迟/抖动 + 直觉评价）
- [ ] 用于判断"我直连这台远程机器的体验如何"

**③ 节点 ↔ 节点测速（有远程节点时）:**
- [ ] 远程节点详情页额外显示 [测试节点间速度] 按钮
- [ ] 由本地节点发起，测试本地与远程节点之间的吞吐量 (iperf3)
- [ ] 结果: 下载 / 上传 + 直觉评价
- [ ] tooltip: "节点之间的直连速度，影响跨节点部署和数据同步的性能"
- [ ] 测速过程通过 SSE 推送实时进度

**前端交互:**
```
--- 本地节点详情页 ---
网络测速                                    [开始测速]
你到这台服务器的速度

┌────────────┐ ┌────────────┐ ┌──────────┐ ┌──────────┐

│  856 Mbps  │ │  421 Mbps  │ │  3.2 ms  │ │  0.8 ms  │
│  非常快    │ │  很快      │ │  极低    │ │  很稳定  │
└────────────┘ └────────────┘ └──────────┘ └──────────┘
上次测速: 2026-03-12 10:30


--- 远程节点详情页 ---

┌────────────┐ ┌────────────┐ ┌──────────┐ ┌──────────┐
│ ↓ 下载     │ │ ↑ 上传     │ │ 延迟  ⓘ │ │ 抖动  ⓘ │
│  120 Mbps  │ │   45 Mbps  │ │  180 ms  │ │  12 ms   │
│  很快      │ │  一般      │ │  较高    │ │  一般    │
└────────────┘ └────────────┘ └──────────┘ └──────────┘

节点间速度  ⓘ                          [测试节点间速度]
```

**技术实现:**
- 浏览器测速: Go 内置 HTTP 端点 (download 返回随机数据流, upload 接收数据计算速度, ping 返回时间戳)
- 节点间测速: iperf3 (每个 Passim 内置 iperf3 server, 监听 :5201)

## US-7.3 SSL 证书状态 `P2` `Phase 4`

**作为** 用户
**我想** 知道 HTTPS 是不是正常的
**以便** 确保浏览器访问不报警告

普通用户不需要理解 SSL/证书的细节，只需要知道"安全连接是否正常"。

**验收标准:**

**状态展示:**
- [ ] 节点详情页 → "安全连接" 区域
- [ ] 用简单语言显示状态:
  - 🟢 "安全连接正常" — 证书有效，自动续期中
  - 🟡 "证书即将到期" — 不到 7 天到期（tooltip: "通常会自动续期，如果持续显示请检查"）
  - 🔴 "证书已过期" — 浏览器会显示安全警告
  - 🔒 "使用自签证书" — (tooltip: "自签证书在局域网使用正常，但浏览器会提示不安全")
  - ⬜ "未配置" — 还没有证书

**详情展开（渐进呈现）:**
- [ ] 点击展开显示技术详情:
  - 证书模式: 自动 (Let's Encrypt) / 自签 / 自定义
  - 域名
  - 颁发者
  - 到期日期
  - 自动续期状态
- [ ] auto 模式: [手动续期] 按钮（通常不需要，但异常时可以手动触发）
- [ ] custom 模式: [上传新证书] 按钮

**主动提醒:**
- [ ] 证书即将过期（< 7 天）且自动续期失败时，Dashboard 显示黄色警告横幅
- [ ] 证书已过期时，Dashboard 显示红色警告横幅

**前端交互:**
```
安全连接

┌──────────────────────────────────────────────────────┐
│ 🟢 安全连接正常                          [展开 ▼]   │
│    证书将于 2026-06-15 到期，届时会自动续期         │
└──────────────────────────────────────────────────────┘

--- 展开后 ---
│ 🟢 安全连接正常                          [收起 ▲]   │
│    证书将于 2026-06-15 到期，届时会自动续期         │
│                                                      │
│    模式: 自动 (Let's Encrypt)                        │
│    域名: abc123.dns.passim.io                        │
│    颁发者: Let's Encrypt                             │
│    到期: 2026-06-15                                  │
│    自动续期: ✅ 已启用                                │
│                                                      │
│    [手动续期]                                        │

--- 证书异常时 Dashboard 横幅 ---
┌──────────────────────────────────────────────────────┐
│ ⚠ tokyo-1 的安全证书将在 3 天后到期，自动续期失败   │
│   [查看详情]                                [忽略]   │
└──────────────────────────────────────────────────────┘
```

**技术实现:**
- certmagic 内置 ACME 客户端，自动管理 Let's Encrypt 证书
- auto 模式: 到期前 30 天自动续期
- self-signed 模式: Go 内置生成自签证书
- custom 模式: 用户上传自己的证书

## US-7.4 DNS 健康检查 `P2` `Phase 4`

**作为** 用户
**我想** 知道域名解析是否正常
**以便** 确保别人能通过域名访问我的服务

**验收标准:**

**状态展示:**
- [ ] 节点详情页 → "域名解析" 区域
- [ ] 简单状态:
  - 🟢 "域名解析正常" — 解析结果与节点 IP 匹配
  - 🔴 "域名解析异常" — 解析 IP 不匹配或解析失败
- [ ] 显示当前域名（如 `abc123.dns.passim.io`）

**详情展开:**
- [ ] 点击展开显示:
  - 期望 IP vs 实际解析 IP
  - 解析失败时的排查建议（用人话）:
    - "DNS 服务可能暂时不可用，通常会自动恢复"
    - "如果持续异常，请检查网络连接"

**前端交互:**
```
域名解析

┌──────────────────────────────────────────────────────┐
│ 🟢 域名解析正常                          [展开 ▼]   │
│    abc123.dns.passim.io → 1.2.3.4                    │
└──────────────────────────────────────────────────────┘

--- 异常时 ---
│ 🔴 域名解析异常                          [展开 ▼]   │
│    abc123.dns.passim.io 解析失败                     │
│                                                      │
│    可能原因:                                         │
│    · DNS 服务暂时不可用，通常会自动恢复              │
│    · 如果持续异常，请检查网络连接                    │
└──────────────────────────────────────────────────────┘
```
