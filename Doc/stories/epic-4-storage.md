# Epic 4: 云硬盘

> Phase: 2 (P2)

云硬盘是 Passim 的外接存储——就像给电脑插 U 盘。用户连接自己的云存储服务（如 AWS S3、Cloudflare R2、MinIO），Passim 里的应用就可以用它来存文件、做备份。

**设计原则：**
- 用"云硬盘"代替"S3 凭证"——普通人理解硬盘，不理解 S3
- 云硬盘本身只是"插上"，用来做什么由应用决定（文件管理器存文件、备份服务存备份等）
- 技术细节（Endpoint、Bucket、Access Key）通过渐进呈现，默认只显示名称和状态
- 常见服务商提供预设，减少手动填写

---

## US-4.1 管理云硬盘 `P2` `Phase 2`

**作为** 用户
**我想** 连接我的云存储，像插硬盘一样
**以便** Passim 里的应用可以用它来存文件或备份

**验收标准:**

**添加云硬盘:**
- [ ] Settings → 云硬盘 → [+ 添加云硬盘] 按钮
- [ ] 第一步选择服务商（预设 + 自定义）:
  - AWS S3
  - Cloudflare R2
  - Backblaze B2
  - MinIO（自建）
  - 其他（自定义）
- [ ] 选择预设后自动填充 Endpoint，减少手动输入
- [ ] 填写表单:
  - 名称（如 "我的备份盘"），placeholder: "给这个硬盘起个名字"
  - Endpoint (带 tooltip: "云存储服务的地址，选了服务商会自动填好")
  - Bucket (带 tooltip: "存储桶，类似硬盘里的一个文件夹")
  - Access Key (带 tooltip: "访问密钥，在云存储服务商后台获取")
  - Secret Key (带 tooltip: "私密密钥，和访问密钥配对使用")
- [ ] [测试连接] 按钮，验证填写的信息能否连通:
  - 成功: "✅ 连接成功，可用空间 XX GB"（如果服务商支持返回容量）
  - 失败: 用人话说原因（"密钥不对" / "找不到这个存储桶" / "网络连不上"）
- [ ] 保存后回到列表

**云硬盘列表:**
- [ ] 每个云硬盘显示:
  - 名称 + 服务商图标
  - 状态指示灯（🟢 可用 / 🔴 连接失败 / 🟡 未测试）
  - 正在使用此硬盘的应用数量（如 "2 个应用正在使用"）
- [ ] 没有云硬盘时显示空状态:
  - 插图 + "还没有连接云硬盘"
  - 说明: "连接云存储后，应用可以用它来存文件和备份"
  - [+ 添加云硬盘] 按钮

**渐进呈现:**
- [ ] 默认列表只显示名称、状态、使用情况
- [ ] 点击展开显示技术详情: Endpoint、Bucket、Access Key（Secret Key 始终隐藏）
- [ ] 展开区域显示 [编辑] [重新测试] [删除] 操作

**编辑与删除:**
- [ ] 编辑: 可修改所有字段，修改后自动重新测试连接
- [ ] 删除前检查是否有应用正在使用:
  - 有应用使用: "「文件管理器」和「自动备份」正在使用这个硬盘，删除后这些应用将无法访问存储"
  - 无应用使用: 普通确认即可

**数据来源:** `GET /api/storage`, `POST /api/storage`, `POST /api/storage/:id/test`
