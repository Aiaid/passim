# Epic 3: 应用

> Phase: 1 (P0) + Phase 2 (P1) + Phase 3 (P1/P2) + Phase 4 (P2)

应用是 Passim 的核心价值——用户来就是为了装 VPN、网盘、远程桌面。体验上对标应用商城：浏览 → 了解 → 一键部署 → 拿到配置 → 分享给朋友。

## US-3.2 应用详情页 `P0` `Phase 2`

**作为** 用户
**我想** 了解一个应用是什么、怎么用、有什么限制
**以便** 决定要不要部署，部署了之后知道怎么用

应用详情页是用户了解和管理一个应用的唯一入口。所有操作（部署、配置、分享、卸载）都从这里发起。

**验收标准:**

**应用信息（所有应用通用，数据来自模板 YAML）:**
- [ ] 顶部: 图标 + 名称 + 分类标签 (Badge)
- [ ] 简介: 2-3 句话说明这是什么、适合什么场景
- [ ] 来源信息:
  - 项目链接（如 GitHub 仓库）
  - 开源协议（如 MIT / GPL / Apache 2.0）
  - 镜像来源（如 linuxserver/wireguard）
  - 维护者
- [ ] 使用指南: 部署后怎么在客户端连接（图文步骤）
  - 支持的客户端 (iOS / Android / Windows / macOS / Linux)
  - 每个平台的下载链接和配置方法
- [ ] 限制说明（如果有）:
  - 如 L2TP: "部分网络环境可能被屏蔽"
  - 如 RDesktop: "需要较多内存 (建议 1GB+)"
  - 如 WireGuard: "iOS/Android 需要安装 WireGuard 客户端"

**部署状态（已部署时显示）:**
- [ ] 部署节点列表，每个节点用颜色标识:
  - 🟢 绿色 — 运行中
  - 🔴 红色 — 已停止 / 异常
  - 🟡 黄色 — 部署中
  - 灰色 — 离线
- [ ] 节点名称 + 简要状态（运行时长 / 错误原因）
- [ ] 单机用户只有一个"本机"节点，不显示节点选择器

**操作区域:**
- [ ] 未部署: [部署] 按钮 (→ US-3.3)
- [ ] 已部署: [配置] [分享] [卸载] 按钮
- [ ] 有远程节点时: [部署到其他节点] (→ US-3.7)

## US-3.3 部署应用 `P0` `Phase 1+2`

**作为** 用户
**我想** 一键把应用部署到我的服务器
**以便** 尽快用上服务

大部分应用只需要极少的配置（WireGuard 就一个 peers 数量）。默认值应该够用，高级选项收起来。

**验收标准:**

**简化部署流程:**
- [ ] 从应用详情页点击 [部署]
- [ ] 如果应用有参数:
  - 常用参数直接显示（如 peers 数量），预填默认值
  - 高级参数折叠在 "高级设置" 中
- [ ] 如果没有参数: 直接确认部署
- [ ] 单机: 直接部署到本机
- [ ] 有远程节点时: 显示目标选择器（本机 + 远程节点），默认选本机
- [ ] 参数验证使用动态构建的 Zod schema
- [ ] 应用配置保存在 UI 所在节点的数据库，然后推送到目标节点执行

**部署进度:**
- [ ] 点击 [部署] 后，按钮变为进度指示，留在当前页面
- [ ] 进度步骤:
  1. 下载镜像 (显示百分比)
  2. 创建服务
  3. 启动服务
  4. 检查运行状态
- [ ] 部署完成后:
  - 页面自动刷新，显示部署状态为 🟢
  - 弹出提示: "部署成功！查看配置以连接" [查看配置]
  - 对于 VPN 类应用: 直接展开配置区域，显示二维码（最高优先级的用户动作）
- [ ] 部署失败: 显示错误原因 + [重试] 按钮

**动态表单字段类型映射:**

| 模板 type | UI 组件 | 说明 |
|----------|---------|------|
| `number` | Slider 或 Stepper | 有 min/max 时用 Slider |
| `string` | Input | |
| `text` | Textarea | 多行文本 |
| `boolean` | Switch | |
| `select` | Select | options 来自模板 |
| `password` | Input (type=password) + 显示/隐藏 | |

## US-3.4 客户端配置 `P0` `Phase 2`

**作为** 用户
**我想** 获取已部署应用的客户端配置
**以便** 在我的设备上连接

这是用户部署应用后最重要的一步。配置页面要尽可能降低"拿到配置 → 连上"的摩擦。

**验收标准:**

**配置展示（按应用类型）:**

| 应用 | 配置方式 |
|------|---------|
| WireGuard | 配置文件列表 (peer1.conf, peer2.conf...) + 二维码 |
| L2TP | 连接参数表格 (服务器/用户名/密码/PSK) + iOS 描述文件 (.mobileconfig) |
| Hysteria | 配置文件 (passim.yaml / stash.yaml) + URL scheme 链接 |
| V2ray | 配置文件 + 订阅链接 + 二维码 |
| WebDAV | 连接地址 + 用户名/密码 + 各平台连接方法 |
| RDesktop | RDP 连接地址 + 端口 + 客户端下载链接 |

**交互:**
- [ ] 每个配置文件/参数旁: [下载] [复制] [二维码]
- [ ] 二维码点击 → 全屏 Dialog 展示，自动调高亮度 (方便手机扫)
- [ ] 有远程节点时: 顶部节点选择器，切换查看不同节点的配置
- [ ] 参数表格中的敏感字段 (密码/PSK) 默认隐藏，点击显示
- [ ] 每个配置旁显示用途说明（tooltip）: "这个配置给你的 iPhone 用" 或 "peer1 = 第 1 台设备的配置"

**前端交互:**
```

## US-3.5 分享给朋友 `P1` `Phase 2`

**作为** 用户
**我想** 把 VPN 配置和使用方法分享给朋友
**以便** 他们不用自己折腾也能用上

这是 Passim 的亮点功能——用户部署了 VPN 后，最常见的下一步就是"怎么给朋友也用上"。

**验收标准:**

**生成分享页面:**
- [ ] 应用详情页 [分享给朋友] 按钮
- [ ] 选择要分享的配置（如 peer3.conf — 给朋友用的那个）
- [ ] 生成一个独立的分享页面 URL
- [ ] 可设置:
  - 有效期（1 小时 / 1 天 / 7 天 / 永不过期）
  - 访问密码（可选）
  - 访问次数限制（可选，如"只能打开 3 次"）

**分享页面内容（朋友打开看到的）:**
- [ ] 干净的独立页面，不需要登录
- [ ] 标题: "你的朋友给你分享了一个 VPN 配置"
- [ ] 配置文件下载 + 二维码
- [ ] 使用指南（和应用详情页相同的图文步骤）:
  - 自动检测访问设备 (iOS/Android/桌面)，优先展示对应平台的指南
  - 其他平台折叠显示
- [ ] 客户端下载链接 (App Store / Play Store / 官网)

**分享方式:**
- [ ] 复制链接
- [ ] 生成二维码（线下场景，朋友用手机扫）
- [ ] 系统分享 (调用浏览器 Share API → 微信/Telegram/iMessage 等)

**安全:**
- [ ] 分享链接使用随机 token，不可猜测
- [ ] 过期后页面显示 "链接已过期"
- [ ] 设置了密码时，先输入密码才能查看
- [ ] 后端 API:
  ```
  POST /api/shares              → { url, token, expires_at }
  GET  /api/shares/:token       → 分享页面内容 (无需认证)
  GET  /api/shares              → 我创建的分享列表
  DELETE /api/shares/:token     → 撤销分享
  ```

## US-3.8 卸载应用 `P2` `Phase 2`

**作为** 用户
**我想** 卸载不再需要的应用
**以便** 释放资源

**验收标准:**

**单节点卸载:**
- [ ] 应用详情页，节点状态列表中每个节点有 [卸载] 操作
- [ ] 确认弹窗: "从 tokyo-1 卸载 WireGuard？配置文件将被删除。"

**完全卸载:**
- [ ] 应用详情页 [卸载 ▼] → "从所有节点卸载"
- [ ] 确认弹窗，显示:
  - 将从 N 个节点卸载
  - 所有客户端配置将失效
  - 所有分享链接将失效
- [ ] 卸载清理: 停止容器 → 删除容器 → 清理配置文件和数据
- [ ] 卸载后返回应用商城页

**前端交互:**
```
[卸载 ▼] → DropdownMenu:
  从 tokyo-1 卸载
  从 us-west 卸载
  ──────────
  从所有节点卸载

--- 完全卸载确认 ---
AlertDialog:
  "卸载 WireGuard"

  将从 2 个节点卸载并删除所有配置。
  3 个客户端配置将失效。

## US-3.10 自定义应用 `P2` `Phase 3`

**作为** 用户
**我想** 部署一个不在应用商城里的 Docker 应用
**以便** 运行自己需要的服务，不受限于内置模板

应用商城覆盖最常用的场景，但 Power User 可能想跑 Minecraft 服务器、Gitea、Home Assistant 等。自定义应用通过向导 UI 帮用户生成标准模板 YAML（spec-templates.md 定义的格式），生成后与内置模板走同一套部署引擎。不是简单的 docker-compose 界面，而是完整的 Passim 模板创建工具。

**验收标准:**

**入口:**
- [ ] 应用商城页面底部: "没找到想要的？[创建自定义应用]"
- [ ] 进入自定义应用向导

**配置向导 (分步):**

**Step 1 — 基本信息:**
- [ ] 应用名称（如 "Minecraft"）
- [ ] Docker 镜像（如 `itzg/minecraft-server:latest`）
  - 输入时实时检查镜像是否存在（`docker pull` 检测）
  - 搜索建议: 输入关键词 → 搜索 Docker Hub（可选，Phase 4）
- [ ] 图标选择（从 lucide 图标库选，或用默认图标）

**Step 2 — 端口与存储:**
- [ ] 端口映射: [+ 添加端口]
  - 主机端口 / 容器端口 / 协议 (TCP/UDP)
  - tooltip: "端口让外部网络能连接到这个服务"
- [ ] 数据目录: [+ 添加目录]
  - 容器内路径 / 说明
  - tooltip: "挂载的目录在容器删除后数据仍然保留"

**Step 3 — 配置项:**
- [ ] 配置方式选择:
  - 环境变量（最常用）: [+ 添加] → KEY / VALUE
  - 配置文件: 文件路径 + 内容编辑器
  - 命令行参数: 自由输入
- [ ] 常见环境变量提示（如 TZ, PUID, PGID）
- [ ] 高级选项（默认折叠）:
  - 特权模式 / capabilities
  - 网络模式 (bridge/host)
  - 重启策略
  - 健康检查命令

**Step 4 — 高级（可选）:**
- [ ] 客户端配置提取（`clients` 块）:
  - 类型: 每用户文件 / 凭证 / URL（默认无）
  - 配置文件路径或凭证字段
  - tooltip: "如果你的应用需要给客户端提供连接信息，在这里配置"
- [ ] 用户管理方式（`users` 块）:
  - 增删用户的方法: 环境变量重启 / 容器内命令 / 配置文件编辑
  - tooltip: "如果你的应用支持多用户，在这里配置怎么增删用户"
- [ ] 用量追踪（`metrics` 块）:
  - 数据来源: 命令 / API / 日志 / 无
- [ ] 以上全部可跳过，之后再补

**Step 5 — 预览与部署:**
- [ ] 显示生成的模板 YAML 预览（可切换可读视图 / 原始 YAML）
- [ ] 高级用户可直接编辑 YAML
- [ ] [部署] 按钮
- [ ] 部署进度复用 US-3.3 的 UI

**部署后:**
- [ ] 自定义应用出现在"应用"列表中，标记为 "自定义"
- [ ] 支持启停、重启、查看日志
- [ ] 支持修改配置（回到向导编辑）
- [ ] [保存为模板] 按钮 (→ US-3.11)

## US-3.11 模板分享与导入 `P2` `Phase 3`

**作为** 用户
**我想** 把我配置好的应用保存为模板，分享给其他 Passim 用户
**以便** 别人不用从头配置，一键就能部署

这是 Passim 的社区生态基础——用户创建的自定义应用可以变成可分享的模板，其他人导入后就像内置应用一样使用。

**验收标准:**

**保存为模板:**
- [ ] 自定义应用详情页 → [保存为模板]
- [ ] 已部署的内置应用也可以 [导出为模板]（携带当前配置）
- [ ] 填写模板信息:
  - 模板名称
  - 一句话描述
  - 分类（VPN / 存储 / 工具 / 游戏 / 其他）
  - 使用指南（可选，Markdown）
  - 哪些配置项需要用户填写 vs 固定值
- [ ] 生成标准模板 YAML（符合 spec-templates.md 格式）
- [ ] 保存到本地模板库

**分享模板:**
- [ ] 模板管理页面（Settings → 我的模板）
- [ ] 分享方式:
  - 导出为 YAML 文件 → 下载
  - 生成分享链接 → 其他 Passim 用户访问后一键导入
  - 复制模板 YAML 到剪贴板
- [ ] 分享链接可设置有效期

**导入模板:**
- [ ] 应用商城页面: [导入模板]
- [ ] 导入方式:
  - 粘贴 YAML 内容
  - 上传 YAML 文件
  - 输入分享链接
- [ ] 导入后校验模板格式
- [ ] 预览模板信息（名称、描述、镜像、配置项）
- [ ] 确认导入 → 出现在应用商城 "自定义" 分区
- [ ] 导入的模板和内置模板体验一致（一键部署、配置表单、客户端配置提取等）

**安全检查:**
- [ ] 导入时警告: "此模板来自社区，请确认 Docker 镜像来源可靠"
- [ ] 显示镜像来源（Docker Hub 链接）
- [ ] 如果模板包含特权模式或 host 网络，额外警告
