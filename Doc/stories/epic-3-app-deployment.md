# Epic 3: 应用

> Phase: 1 (P0) + Phase 2 (P1) + Phase 3 (P1/P2) + Phase 4 (P2)

应用是 Passim 的核心价值——用户来就是为了装 VPN、网盘、远程桌面。体验上对标应用商城：浏览 → 了解 → 一键部署 → 拿到配置 → 分享给朋友。

## US-3.2 应用详情页 `P0` `Phase 2`

**作为** 用户
**我想** 了解一个应用是什么、怎么用、有什么限制
**以便** 决定要不要部署，部署了之后知道怎么用

应用详情页是用户了解和管理一个应用的唯一入口。所有操作（部署、配置、分享、卸载）都从这里发起。

**验收标准:**

**应用信息（所有应用通用，数据来自模板 YAML）:**
- [ ] 顶部: 图标 + 名称 + 分类标签 (Badge)
- [ ] 简介: 2-3 句话说明这是什么、适合什么场景
- [ ] 来源信息:
  - 项目链接（如 GitHub 仓库）
  - 开源协议（如 MIT / GPL / Apache 2.0）
  - 镜像来源（如 linuxserver/wireguard）
  - 维护者
- [ ] 使用指南: 部署后怎么在客户端连接（图文步骤）
  - 支持的客户端 (iOS / Android / Windows / macOS / Linux)
  - 每个平台的下载链接和配置方法
- [ ] 限制说明（如果有）:
  - 如 L2TP: "部分网络环境可能被屏蔽"
  - 如 RDesktop: "需要较多内存 (建议 1GB+)"
  - 如 WireGuard: "iOS/Android 需要安装 WireGuard 客户端"

**部署状态（已部署时显示）:**
- [ ] 部署节点列表，每个节点用颜色标识:
  - 🟢 绿色 — 运行中
  - 🔴 红色 — 已停止 / 异常
  - 🟡 黄色 — 部署中
  - 灰色 — 离线
- [ ] 节点名称 + 简要状态（运行时长 / 错误原因）
- [ ] 单机用户只有一个"本机"节点，不显示节点选择器

**操作区域:**
- [ ] 未部署: [部署] 按钮 (→ US-3.3)
- [ ] 已部署: [配置] [分享] [卸载] 按钮
- [ ] 有远程节点时: [部署到其他节点] (→ US-3.7)

## US-3.3 部署应用 `P0` `Phase 1+2`

**作为** 用户
**我想** 一键把应用部署到我的服务器
**以便** 尽快用上服务

大部分应用只需要极少的配置（WireGuard 就一个 peers 数量）。默认值应该够用，高级选项收起来。

**验收标准:**

**简化部署流程:**
- [ ] 从应用详情页点击 [部署]
- [ ] 如果应用有参数:
  - 常用参数直接显示（如 peers 数量），预填默认值
  - 高级参数折叠在 "高级设置" 中
- [ ] 如果没有参数: 直接确认部署
- [ ] 单机: 直接部署到本机
- [ ] 有远程节点时: 显示目标选择器（本机 + 远程节点），默认选本机
- [ ] 参数验证使用动态构建的 Zod schema
- [ ] 应用配置保存在 UI 所在节点的数据库，然后推送到目标节点执行

**部署进度:**
- [ ] 点击 [部署] 后，按钮变为进度指示，留在当前页面
- [ ] 进度步骤:
  1. 下载镜像 (显示百分比)
  2. 创建服务
  3. 启动服务
  4. 检查运行状态
- [ ] 部署完成后:
  - 页面自动刷新，显示部署状态为 🟢
  - 弹出提示: "部署成功！查看配置以连接" [查看配置]
  - 对于 VPN 类应用: 直接展开配置区域，显示二维码（最高优先级的用户动作）
- [ ] 部署失败: 显示错误原因 + [重试] 按钮

**动态表单字段类型映射:**

| 模板 type | UI 组件 | 说明 |
|----------|---------|------|
| `number` | Slider 或 Stepper | 有 min/max 时用 Slider |
| `string` | Input | |
| `text` | Textarea | 多行文本 |
| `boolean` | Switch | |
| `select` | Select | options 来自模板 |
| `password` | Input (type=password) + 显示/隐藏 | |

## US-3.4 客户端配置 `P0` `Phase 2`

**作为** 用户
**我想** 获取已部署应用的客户端配置
**以便** 在我的设备上连接

这是用户部署应用后最重要的一步。配置页面要尽可能降低"拿到配置 → 连上"的摩擦。

**验收标准:**

**配置展示（按应用类型）:**

| 应用 | 配置方式 |
|------|---------|
| WireGuard | 配置文件列表 (peer1.conf, peer2.conf...) + 二维码 |
| L2TP | 连接参数表格 (服务器/用户名/密码/PSK) + iOS 描述文件 (.mobileconfig) |
| Hysteria | 配置文件 (passim.yaml / stash.yaml) + URL scheme 链接 |
| V2ray | 配置文件 + 订阅链接 + 二维码 |
| WebDAV | 连接地址 + 用户名/密码 + 各平台连接方法 |
| RDesktop | RDP 连接地址 + 端口 + 客户端下载链接 |

**交互:**
- [ ] 每个配置文件/参数旁: [下载] [复制] [二维码]
- [ ] 二维码点击 → 全屏 Dialog 展示，自动调高亮度 (方便手机扫)
- [ ] 有远程节点时: 顶部节点选择器，切换查看不同节点的配置
- [ ] 参数表格中的敏感字段 (密码/PSK) 默认隐藏，点击显示
- [ ] 每个配置旁显示用途说明（tooltip）: "这个配置给你的 iPhone 用" 或 "peer1 = 第 1 台设备的配置"

**前端交互:**
```

## US-3.5 分享给朋友 `P1` `Phase 2`

**作为** 用户
**我想** 把 VPN 配置和使用方法分享给朋友
**以便** 他们不用自己折腾也能用上

这是 Passim 的亮点功能——用户部署了 VPN 后，最常见的下一步就是"怎么给朋友也用上"。

**验收标准:**

**生成分享页面:**
- [ ] 应用详情页 [分享给朋友] 按钮
- [ ] 选择要分享的配置（如 peer3.conf — 给朋友用的那个）
- [ ] 生成一个独立的分享页面 URL
- [ ] 可设置:
  - 有效期（1 小时 / 1 天 / 7 天 / 永不过期）
  - 访问密码（可选）
  - 访问次数限制（可选，如"只能打开 3 次"）

**分享页面内容（朋友打开看到的）:**
- [ ] 干净的独立页面，不需要登录
- [ ] 标题: "你的朋友给你分享了一个 VPN 配置"
- [ ] 配置文件下载 + 二维码
- [ ] 使用指南（和应用详情页相同的图文步骤）:
  - 自动检测访问设备 (iOS/Android/桌面)，优先展示对应平台的指南
  - 其他平台折叠显示
- [ ] 客户端下载链接 (App Store / Play Store / 官网)

**分享方式:**
- [ ] 复制链接
- [ ] 生成二维码（线下场景，朋友用手机扫）
- [ ] 系统分享 (调用浏览器 Share API → 微信/Telegram/iMessage 等)

**安全:**
- [ ] 分享链接使用随机 token，不可猜测
- [ ] 过期后页面显示 "链接已过期"
- [ ] 设置了密码时，先输入密码才能查看
- [ ] 后端 API:
  ```
  POST /api/shares              → { url, token, expires_at }
  GET  /api/shares/:token       → 分享页面内容 (无需认证)
  GET  /api/shares              → 我创建的分享列表
  DELETE /api/shares/:token     → 撤销分享
  ```

## US-3.8 卸载应用 `P2` `Phase 2`

**作为** 用户
**我想** 卸载不再需要的应用
**以便** 释放资源

**验收标准:**

**单节点卸载:**
- [ ] 应用详情页，节点状态列表中每个节点有 [卸载] 操作
- [ ] 确认弹窗: "从 tokyo-1 卸载 WireGuard？配置文件将被删除。"

**完全卸载:**
- [ ] 应用详情页 [卸载 ▼] → "从所有节点卸载"
- [ ] 确认弹窗，显示:
  - 将从 N 个节点卸载
  - 所有客户端配置将失效
  - 所有分享链接将失效
- [ ] 卸载清理: 停止容器 → 删除容器 → 清理配置文件和数据
- [ ] 卸载后返回应用商城页

**前端交互:**
```
[卸载 ▼] → DropdownMenu:
  从 tokyo-1 卸载
  从 us-west 卸载
  ──────────
  从所有节点卸载

--- 完全卸载确认 ---
AlertDialog:
  "卸载 WireGuard"
