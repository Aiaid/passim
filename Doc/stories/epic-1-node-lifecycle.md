# Epic 1: 节点生命周期

> Phase: 1 (P0) + Phase 3 (P1/P2) + Phase 4 (P2)

覆盖一个 Passim 节点从安装到卸载的完整生命周期：安装 → 首次设置 → 日常运行 → 更新 → 远程连接 → 删除/卸载。

---

## US-1.1 安装 Passim `P0` `Phase 1`

**作为** 有 VPS 的人（或帮他安装的朋友）
**我想** 在 VPS 上一行命令启动 Passim
**以便** 之后在浏览器里管理这台机器，不再碰终端

安装是整个流程中唯一需要命令行的步骤。之后所有操作都在 Web UI 完成。

**验收标准:**
- [ ] 一行命令启动:
  ```bash
  docker run -d --name passim --restart always \
    -v /var/run/docker.sock:/var/run/docker.sock \
    -v passim-data:/data \
    -p 8443:8443 -p 80:80 \
    passim/passim:latest
  ```
  端口 80 用于 ACME HTTP-01 证书验证 (auto 模式)
- [ ] 首次启动自动生成 API Key，输出到 Docker 日志 (`docker logs passim`)
- [ ] 首次启动自动运行 setup: 初始化 SSL 证书 (certmagic) + 内置测速服务
- [ ] 启动完成后可通过 `https://<vps-ip>:8443` 访问 Web UI
- [ ] 启动时间 < 5s (不含 setup)
- [ ] 支持通过环境变量自定义节点名和 API Key:
  ```bash
  -e PASSIM_NODE_NAME="tokyo-1"
  -e PASSIM_AUTH_API_KEY="my-custom-key"
  ```

## US-1.2 首次设置向导 `P0` `Phase 2`

**作为** 第一次打开 Passim 的用户
**我想** 被引导完成初始设置
**以便** 不用去看 docker logs，不用猜下一步该做什么

首次打开 Web UI 时，不是直接跳登录页，而是进入一个简洁的设置向导。

**验收标准:**
- [ ] 检测到 `setup_complete = false` 时，所有请求重定向到 Welcome 页面
- [ ] 向导步骤:

**Step 1 — 你的 API Key**
- [ ] 全屏显示自动生成的 API Key（大字，显眼）
- [ ] 说明文字: "这是你登录 Passim 的钥匙，请保存好"
- [ ] [复制] 按钮，复制后按钮变为 "已复制"
- [ ] 底部提示: "也可以通过 `docker logs passim` 查看"
- [ ] [我已保存，继续] 按钮

**Step 2 — 注册 Passkey（可选）**
- [ ] 说明: "注册指纹/面容，下次打开直接登录，不用再输 Key"
- [ ] [注册 Passkey] 按钮 → 触发 WebAuthn 注册流程
- [ ] 注册成功显示确认
- [ ] [跳过] 链接，不强制

**Step 3 — 给节点起个名字（可选）**
- [ ] 输入框，placeholder 显示默认名 (主机名 或 "my-server")
- [ ] 说明: "方便你识别这台机器，尤其管理多台时"
- [ ] [跳过] 或 [完成]

- [ ] 完成后标记 `setup_complete = true`，跳转 Dashboard
- [ ] 向导只出现一次，之后正常登录

## US-1.3 查看节点状态 `P0` `Phase 1`

**作为** 用户
**我想** 看到 VPS 的运行状态
**以便** 知道"我的服务器还好吗、空间还够不够"

**验收标准:**

**简洁摘要（默认视图，面向普通用户）:**
- [ ] Dashboard 顶部 4 个指标卡片:
  - CPU 使用率（百分比 + 简洁进度条）
  - 内存（已用 / 总量，如 "5.3 / 8 GB"）
  - 磁盘（已用 / 总量，如 "40 / 100 GB"）
  - 网络（上行/下行实时速率）
- [ ] 已部署的应用列表和运行状态（🟢/🔴）
- [ ] 数据每 5 秒自动刷新 (SSE 推送)

**展开详情（点击卡片或"更多"，面向 Power User）:**
- [ ] CPU 核数、型号
- [ ] 系统负载 (1/5/15 min)
- [ ] 操作系统版本、内核版本
- [ ] 运行中 / 已停止的容器数量
- [ ] 启动时间 (uptime)

## US-1.4 添加远程节点 `P1` `Phase 3`

**作为** 管理多台 VPS 的用户
**我想** 从当前 Passim 连接管理其他 VPS 上的 Passim
**以便** 在一个面板管理多台机器

**验收标准:**
- [ ] Dashboard "远程节点" 区域显示 [+ 添加节点] 按钮
- [ ] 弹窗填写: 地址 (host:port) + 目标 Passim 的 API Key + 可选名称
- [ ] 提交后自动建立 WebSocket 连接
- [ ] 连接成功后节点卡片显示 "🟢 已连接" + 基础指标
- [ ] 连接失败显示具体原因 (网络不通 / API Key 错误 / 超时)
- [ ] 断线自动重连 (指数退避，上限 60s)
- [ ] 没有远程节点时，该区域不显示

**前端交互:**

## US-1.6 删除远程节点 `P1` `Phase 3`

**作为** 用户
**我想** 移除一个不再需要管理的远程节点
**以便** 清理面板，断开不再需要的连接

从 US-1.5 的"可删除"展开为独立 story，因为删除涉及多个决策。

**验收标准:**
- [ ] 在远程节点列表中，每个节点有 [删除] 操作
- [ ] 删除前确认弹窗，显示:
  - 节点名称和地址
  - 该节点上由我远程部署的应用数量（如果有）
- [ ] 删除选项:
  - "仅断开连接" — 断开 WebSocket，从本机列表移除，远程节点上的应用保持运行
  - "断开并卸载我部署的应用" — 同时向远程节点发送卸载指令
- [ ] 删除后从 Dashboard 和 Settings 列表中立即消失
- [ ] 远程节点侧: 连接断开，入站连接列表中该记录消失

**前端交互:**
```
AlertDialog: 删除远程节点 "tokyo-2"

  该节点上有 2 个由你部署的应用 (WireGuard, WebDAV)

  ○ 仅断开连接 (应用保持运行)
  ○ 断开并卸载我部署的应用

  [取消]                         [删除]
```

## US-1.8 更新 Passim `P2` `Phase 4`

**作为** 用户
**我想** 知道有没有新版本，能简单地更新

**版本显示:**
- [ ] 侧边栏底部显示当前版本号 (如 `v1.0.0`)
- [ ] Settings → 关于 页面显示: 版本号、构建时间、节点 ID (可复制)

**更新检查:**
- [ ] 启动后自动检查更新，之后每 24 小时检查一次 (`GET /api/version/check`)
- [ ] 有新版本时，侧边栏版本号旁显示小圆点提示
- [ ] Settings → 关于 显示: 新版本号 + 更新说明 (changelog 摘要)
- [ ] 无新版本时显示 "已是最新版本 ✓"
- [ ] 检查失败静默处理，不打扰用户

**一键更新:**
- [ ] [更新到 vX.X.X] 按钮

## US-1.9 卸载 Passim `P2` `Phase 4`

**作为** 用户
**我想** 彻底从 VPS 上移除 Passim
**以便** 不再使用时干净清理

卸载需要回到命令行 (或通过 Web UI 触发)。关键是要清理干净，不留残余。

**验收标准:**

**Web UI 引导卸载:**
- [ ] Settings → 关于 页面底部 [卸载 Passim] 按钮 (destructive 样式)
- [ ] 确认弹窗，显示:
  - 将要清理的内容: Passim 容器 + 由 Passim 部署的所有应用容器 + 数据卷
  - 影响: 所有远程连接将断开，所有入站管理连接将断开
- [ ] 选项:
  - ☐ 同时删除所有应用容器和数据 (默认勾选)
  - ☐ 保留应用容器 (仅移除 Passim 自身)
- [ ] 确认后:
  1. 通知所有远程连接的节点 (对方会看到此节点已离线)
  2. 停止并删除由 Passim 部署的应用容器 (如果勾选)
  3. 输出卸载命令到页面 (用户需要手动执行最后一步):
     ```bash
     docker stop passim && docker rm passim
     docker volume rm passim-data
     ```
     (Passim 无法删除自己正在运行的容器)

**CLI 卸载 (替代方案):**
- [ ] 支持 `docker exec passim passim uninstall` 命令
- [ ] 该命令清理所有应用容器和数据卷，然后输出删除自身的命令

**前端交互:**
```

│                                              │
│  [ 卸载 Passim ]                             │
│                                              │

  这将:
  · 断开所有远程节点连接
  · 停止并删除 3 个应用容器 (WireGuard, L2TP, WebDAV)
  · 清除所有配置和数据

  ☑ 同时删除所有应用容器和数据
  ☐ 保留应用容器 (仅移除 Passim)

  [取消]                    [确认卸载]

--- 卸载完成 ---

  ✅ 应用容器已清理

  最后一步，在终端运行:
  ┌──────────────────────────────────────┐
  │ docker stop passim && docker rm p... │ [复制]
```
