# Epic 2: 容器管理（系统监视器）

> Phase: 1 (P0) + Phase 4 (P2)

容器管理是 Passim 的"任务管理器"——大部分时候用户通过"应用"页面管理服务，但需要深入排查或了解资源占用时，可以到这里看底层情况。

**设计原则（对齐 UniFi 风格）：**
- 侧边栏中排在 Dashboard、Apps 之后，不是首要入口
- 默认视图简洁直观，技术细节通过 hover/展开渐进呈现
- 所有专业术语配 tooltip 解释（如 hover "容器" → "容器是运行中的服务实例，类似一个独立运行的小程序"）
- 应用容器点击优先跳到应用详情页，不是停留在容器层面
- 状态用颜色和图标表达，不用 running/stopped 这样的文字

---

## US-2.1 查看容器列表 `P0` `Phase 1`

**作为** 用户
**我想** 看到服务器上所有运行中的服务和它们的资源占用
**以便** 了解什么在跑、占了多少资源

**验收标准:**

**列表展示:**
- [ ] 每行显示: 名称、状态指示灯（🟢 运行中 / 🔴 已停止 / 🟡 启动中）、运行时长、CPU%、内存占用
- [ ] Passim 部署的应用容器和用户自建的容器分组显示:
  - "应用" 分组 — 通过 Passim 部署的，名称旁带应用图标
  - "其他" 分组 — 用户自己 docker run 的
- [ ] 应用容器行点击 → 跳转到对应的应用详情页（而不是容器详情）
- [ ] Passim 自身容器不显示在列表中

**渐进呈现（Progressive Disclosure）:**
- [ ] 默认隐藏技术细节（镜像名、端口映射、容器 ID）
- [ ] 鼠标 hover 容器行 → 显示简要信息 tooltip:
  - 应用容器: "WireGuard VPN — 由 Passim 部署，运行了 3 天"
  - 外部容器: "nginx:alpine — 非 Passim 管理的容器"
- [ ] 点击展开行 → 显示详细信息:
  - 镜像名 (带 tooltip: "镜像是这个服务的安装包")
  - 端口映射 (带 tooltip: "外部端口 → 容器端口，用于网络访问")
  - 创建时间
  - 容器 ID（可复制，用于高级排查）

**实时更新:**
- [ ] 状态和资源数据通过 SSE 实时更新
- [ ] 容器启停时列表自动刷新（动画过渡，不闪烁）

**过滤:**
- [ ] 支持按名称搜索
- [ ] 状态筛选: 全部 / 运行中 / 已停止

**数据来源:** `GET /api/containers`

**前端交互:**

## US-2.2 容器操作 `P0` `Phase 1`

**作为** 用户
**我想** 启动、停止、重启或删除一个服务
**以便** 管理服务器上的服务运行状态

**验收标准:**

**操作菜单:**
- [ ] 每行右侧 ⋮ 按钮，展开操作菜单
- [ ] 操作项使用易懂的语言 + 图标:
  - ▶ 启动 (已停止时)
  - ⏸ 停止 (运行中时)，tooltip: "暂停这个服务，不会删除数据"
  - 🔄 重启 (运行中时)，tooltip: "重新启动，通常用于修复卡住的服务"
  - 🗑 删除，tooltip: "彻底移除这个服务和它的配置"
- [ ] 应用容器的操作菜单额外显示: "查看应用详情" 跳转链接

**安全确认:**
- [ ] 停止和重启: 直接执行（可撤销，低风险）
- [ ] 删除: 二次确认弹窗
  - 应用容器删除时: "这是由 Passim 部署的应用，建议到应用页面卸载以确保完全清理。仍要在这里删除？"
  - 外部容器删除时: "确定删除 my-nginx？"
  - 选项: ☐ 同时删除数据 (tooltip: "勾选后会删除这个服务保存的所有文件")

**反馈:**
- [ ] 操作后即时反馈 (乐观更新 + 服务端确认)
- [ ] 操作进行中行显示 loading 动画
- [ ] 操作失败显示 Toast 错误提示，用普通人能懂的语言:
  - ~~"port conflict"~~ → "端口被占用，另一个服务正在使用相同的端口"
  - ~~"image not found"~~ → "找不到服务的安装包，可能已被删除"

**前端交互:**
```
⋮ 按钮 → DropdownMenu:
  ▶ 启动
  ⏸ 停止           "暂停服务，数据不会丢失"
  🔄 重启           "重新启动服务"
  ──────────
  📋 查看应用详情    (仅应用容器)
  ──────────
  🗑 删除

--- 删除应用容器时 ---
AlertDialog:
  "删除 WireGuard 容器"

  ⚠ 这是由 Passim 部署的应用。
  建议到「应用」页面卸载，以确保完全清理配置和数据。

  [去应用页面]              [仍然删除]

--- 删除外部容器时 ---
AlertDialog:
  "删除 my-nginx"

  ☐ 同时删除数据  ⓘ

  [取消]                    [删除]
```

## US-2.3 查看容器日志 `P2` `Phase 4`

**作为** 用户
**我想** 查看一个服务的运行日志
**以便** 排查问题或确认服务正常

日志是偏高级的功能，但也是排查问题时最重要的工具。保持简洁，不需要用户理解日志格式。

**验收标准:**

**打开方式:**
- [ ] 容器行 ⋮ 菜单 → "查看日志"
- [ ] 从右侧滑入的面板 (Sheet)，不离开当前页面

**日志展示:**
- [ ] 默认显示最近 200 行
- [ ] 等宽字体，保留颜色（ANSI）
- [ ] 自动滚动到最新内容（可关闭）
- [ ] 新日志实时追加 (SSE 推送)

**搜索与导出:**
- [ ] 顶部搜索框，输入关键词 → 高亮匹配行
- [ ] [下载] 按钮导出完整日志文件
- [ ] 日志级别颜色区分: INFO 灰色 / WARN 黄色 / ERROR 红色

**帮助提示:**
- [ ] 面板顶部小字: "日志记录了服务的运行过程。如果服务出了问题，这里通常能找到原因。"
- [ ] 搜索框 placeholder: "搜索日志内容，如 error、timeout..."

**前端交互:**
```
Sheet (从右侧滑入, 宽度 50%):
┌─────────────────────────────────────────────────────┐
│  WireGuard 日志                                [×]  │
│  日志记录了服务的运行过程                           │
│                                                     │
│  [🔍 搜索日志内容...]  [⬇ 下载]  [实时 ☑]         │
│─────────────────────────────────────────────────────│
│  10:00:01  Started                                  │
│  10:00:02  Peer connected: 10.0.0.2                 │
│  10:00:05  ⚠ Handshake timeout for peer 3          │
│  ...                                                │
│  (实时模式: 新日志自动出现在底部)                    │
└─────────────────────────────────────────────────────┘
```
