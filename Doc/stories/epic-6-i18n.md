# Epic 6: 多语言

> Phase: 2 (P1)

Passim 默认中文，支持切换英文。界面语言影响所有 UI 文本、应用描述、使用教程和错误信息。

**设计原则：**
- 中文为默认语言（目标用户群体）
- 语言切换入口不占主要位置——放在侧边栏底部或 Settings 中
- 切换即时生效，无需刷新
- 应用模板的名称、描述、使用指南都要支持多语言

---

## US-6.1 切换语言 `P1` `Phase 2`

**作为** 用户
**我想** 在中文和英文之间切换界面语言
**以便** 使用我熟悉的语言

**验收标准:**

**语言切换入口:**
- [ ] 侧边栏底部，版本号旁边: 语言切换按钮 (如 "中文 / EN")
- [ ] Settings → 通用 中也有语言选择
- [ ] 点击后弹出选项: 中文 / English
- [ ] 切换后立即生效，无需刷新页面

**语言检测与记忆:**
- [ ] 首次访问根据浏览器 `Accept-Language` 自动选择
- [ ] 语言偏好保存到 localStorage，下次访问自动应用
- [ ] 手动选择优先于浏览器检测

**翻译覆盖范围:**
- [ ] 所有 UI 文本（导航、按钮、标签、提示、tooltip）
- [ ] 应用模板名称和描述（YAML 中支持多语言 key）
- [ ] 使用指南 / 客户端配置教程
- [ ] 错误信息和 Toast 提示
- [ ] 时间格式（"3 分钟前" / "3 minutes ago"）
- [ ] 空状态文案和引导提示
- [ ] 分享页面（给朋友的页面也跟随语言设置）

**前端交互:**
```
--- 侧边栏底部 ---
  v1.0.0  ·  [中文 ▼]
               ├── 中文
               └── English

--- Settings → 通用 ---
语言
┌──────────────────────────────────────┐
│  界面语言                            │
│  [中文                          ▼]  │
│                                      │
│  · 中文                              │
│  · English                           │
└──────────────────────────────────────┘
```

**技术实现:**
- `react-i18next` 或轻量 i18n 方案
- 消息文件: `locales/zh-CN.json`, `locales/en-US.json`
- 应用模板 YAML 中 `description`/`label`/`guide` 字段支持多语言 key:
  ```yaml
  label:
    zh-CN: "WireGuard VPN"
    en-US: "WireGuard VPN"
  description:
    zh-CN: "快速安全的点对点 VPN"
    en-US: "Fast and secure point-to-point VPN"
  ```
