# Epic 9: 设置与个性化

> Phase: 2 (P1)

Settings 是所有配置的集中入口。大部分功能在其他 Epic 中定义，Settings 负责把它们组织在一起。

**Settings 页面结构：**
- **通用** — 节点名称 (US-1.7)、语言 (US-6.1)、主题 (US-9.1)
- **安全** — Passkey 管理 (US-5.3)、API Key 管理 (US-5.1)
- **云硬盘** — 云存储连接 (US-4.1)
- **共享管理** — 分享总览 (US-8.1)，Phase 5
- **远程节点** — 出站/入站连接管理 (US-1.5)，Phase 3
- **系统** — 版本信息 + 软件更新 (US-1.8)，已实现 Phase 4；卸载 (US-1.9) 待定

## US-9.1 暗色模式 `P1` `Phase 2`

**作为** 用户
**我想** 切换亮色/暗色主题
**以便** 在不同光线环境下舒适使用

**验收标准:**
- [ ] Settings → 通用 中主题选择（三种模式）:
  - 浅色
  - 深色
  - 跟随系统
- [ ] 侧边栏底部也提供快捷切换图标（🌙/☀️）
- [ ] 切换动画平滑 (CSS transition)
- [ ] 偏好保存到 localStorage，下次访问自动应用
- [ ] 所有页面和组件适配暗色模式 (shadcn/ui 原生支持)
- [ ] 图表和状态颜色在暗色模式下保持可读性
- [ ] 默认跟随系统

**前端交互:**
```
Settings → 通用

主题
┌──────────────────────────────────────┐
│  外观                                │
│  [☀ 浅色]  [🌙 深色]  [💻 跟随系统] │
└──────────────────────────────────────┘

--- 侧边栏底部 ---
  v1.0.0 · 中文 · [🌙]
```

**技术实现:**
- shadcn/ui 通过 CSS 变量 + `class="dark"` 实现
- OKLCH 色彩空间确保亮暗模式颜色感知一致

## US-9.3 系统与软件更新 `P1` `Phase 4`

**作为** 用户
**我想** 查看当前版本信息并检查/安装更新
**以便** 保持系统最新

**验收标准:**
- [x] Settings → 系统 标签页显示版本信息 (版本号、Git commit、构建时间)
- [x] Dev 版本 (`dev` / `dev-<sha>`) 显示"开发版"徽章
- [x] 默认检查稳定版更新 (GitHub Releases `/releases/latest`)
- [x] "包含预发布版本"开关 — 开启后检查 beta/rc 等预发布版本
- [x] 有更新时显示新版本号、发布时间、更新日志 (changelog)，一键安装
- [x] Dev 版本运行时显示"最新发布版"而非"发现新版本"（dev 可能比 release 更新）
- [x] 预发布开关开启后显示"开发版通道"——强制拉取 `ghcr.io/aiaid/passim:dev` (main 分支最新构建)
- [x] 更新过程中显示 loading 状态和断连提示

**前端交互:**
```
Settings → 系统

┌─ 版本信息 ──────────────────────────────┐
│  当前版本    dev-abc1234  [开发版]       │
│  提交        abc1234                     │
│  构建时间    2026/03/20 14:30            │
└─────────────────────────────────────────┘

┌─ 软件更新 ──────────────────────────────┐
│  包含预发布版本              [Switch]    │
│  开启后可获取 dev / beta / rc           │
│  ─────────────────────────────────────  │
│  最新发布版  v1.2.0  [预发布]           │
│  发布于 2026/03/19        [切换到此版本] │
│  ─────────────────────────────────────  │
│  ⑂ 开发版通道            [拉取最新 dev] │
│  强制拉取 main 分支最新构建             │
└─────────────────────────────────────────┘
```

**技术实现:**
- 后端: `GET /api/version` (公开) 返回版本/commit/build_time
- 后端: `GET /api/version/check[?prerelease=true]` 检查 GitHub Releases
- 后端: `POST /api/update {"version":"dev"}` 拉取 `:dev` tag 并执行自更新
- `Checker.CheckPrerelease()` 使用 `/releases` 端点含预发布版本
- Dev 版本通过 `isNewerOrDev()` 对任何正式 release 标记为可安装
- 更新通道 (`prerelease` toggle) 状态为前端本地状态，不持久化
