# Epic 10: 手机 App

> Phase: 5 (P1)

面向普通人的产品，手机才是主场。用户不会天天开电脑看 Dashboard，但手机随时在手边——收到通知看一眼、扫个码连 VPN、出门前确认服务还活着。Passim 手机 App 不是 Web UI 的缩小版，而是围绕"随手管理"这个场景重新设计的。

> 技术详情见 [../spec-app.md](../spec-app.md)

---

## US-10.1 连接 Passim 节点 `P1` `Phase 5`

**作为** 用户
**我想** 在手机上添加我的 Passim 节点
**以便** 随时查看和管理

**验收标准:**
- [ ] 首次打开 App，引导页说明 Passim 是什么 + 如何开始 (1-2 屏)
- [ ] 添加节点方式:
  - 扫码添加: Web UI 设置页生成二维码，App 扫码一步完成（推荐方式）
  - 手动输入: 地址 (host:port) + API Key
- [ ] 添加后自动连接验证:
  - 成功 → 提示注册生物认证 → 进入 Dashboard
  - 失败 → 人话错误（"API Key 不正确" / "连接超时，检查地址是否正确"）
- [ ] 支持添加多个节点，底部 Tab 或顶部横滑切换
- [ ] 节点列表显示: 名称、地址、🟢 在线 / 🔴 离线
- [ ] 离线节点标灰，定期自动重试连接

**前端交互:**

```
│  └───────────────────────┘  │
│                             │
│  ┌───────────────────────┐  │

```

## US-10.3 应用管理与快捷操作 `P1` `Phase 5`

**作为** 用户
**我想** 在手机上启停应用、查看状态
**以便** 不用开电脑也能管理服务

手机上以快捷操作为主（启停、重启、查看配置），完整的部署和配置流程留给 Web UI。

**验收标准:**

**应用列表:**
- [ ] 应用列表页: 所有已部署应用，按节点分组（多节点时）
- [ ] 每个应用显示: 图标、名称、运行状态、节点名

**应用详情:**
- [ ] 运行状态 + 资源占用 (CPU / 内存)
- [ ] 操作按钮: 启动 / 停止 / 重启（确认弹窗）
- [ ] 客户端配置入口 (→ US-10.4)
- [ ] 分享给朋友入口 (→ 生成分享链接，手机上分享比 Web 更自然)

**快捷部署（简化版）:**
- [ ] 应用列表页 [+ 部署] 入口
- [ ] 模板网格（复用 Web 的模板数据 `GET /api/templates`）
- [ ] 参数表单（适配手机屏幕，一屏一个配置项）
- [ ] 部署进度页 (SSE 实时更新)
- [ ] 多节点时可选择部署到哪个节点

## US-10.4 VPN 配置与导入 `P0` `Phase 5`

**作为** 用户
**我想** 在手机上直接获取 VPN 配置并导入客户端
**以便** 不用在电脑和手机之间来回传文件

这是手机 App 最核心的场景——大部分用户装 Passim 就是为了用 VPN，而 VPN 配置的最终消费者就是手机。

**验收标准:**

**WireGuard 配置:**
- [ ] 显示配置文件列表 (peer1.conf, peer2.conf...)
- [ ] 点击 → 弹出选项:
  - "导入到 WireGuard" — 通过系统分享或 deep link 打开 WireGuard App
  - "显示二维码" — 全屏显示，自动调高屏幕亮度（方便别人扫）
  - "分享" — 系统分享面板
  - "复制内容" — 复制配置文本 + 触觉反馈

**L2TP 配置:**
- [ ] 显示连接参数（服务器 / 用户名 / 密码 / PSK）
- [ ] 一键复制各字段
- [ ] iOS: "安装描述文件" 按钮（下载 .mobileconfig，自动配置 VPN）

**Hysteria 配置:**
- [ ] "导入到 Stash/Shadowrocket"（通过 URL scheme）
- [ ] 显示配置内容 + 复制

**通用:**
- [ ] 配置缓存到本地，离线也能查看
- [ ] 多节点场景，顶部切换节点

**前端交互:**

## US-10.5 推送通知 `P2` `Phase 5`

**作为** 用户
**我想** 在手机上收到重要事件通知
**以便** 及时处理问题，不用一直盯着 Dashboard

**验收标准:**

**通知类型:**
- [ ] 节点离线 / 恢复上线
- [ ] 容器异常停止
- [ ] SSL 证书即将到期 (7 天前)
- [ ] 部署完成 / 失败
- [ ] 有新版本 Passim 可用

**通知设置:**
- [ ] 可按类型开关通知
- [ ] 可设置勿扰时段
- [ ] 可按节点单独配置

**交互:**
- [ ] 点击通知 → 跳转到对应页面（节点详情/应用详情）
- [ ] 通知中心页面显示历史通知
- [ ] 实现方式: Expo Push Notifications + Passim API 注册设备 token

**后端支持 (Passim 侧):**
```
POST /api/push/register    { "token": "ExponentPushToken[xxx]", "device": "iPhone 16" }
DELETE /api/push/unregister  { "token": "..." }
GET /api/push/settings     → 通知偏好
PUT /api/push/settings     → 更新通知偏好
```

## US-10.6 Passkey 生物认证 `P1` `Phase 5`

**作为** 用户
**我想** 用指纹或面容登录 App
**以便** 安全又方便，不用每次输 API Key

**验收标准:**
- [ ] 首次添加节点用 API Key 认证 (→ US-10.1)
- [ ] 认证成功后提示注册生物认证: "使用 Face ID / 指纹登录？"
- [ ] 注册后，下次打开 App 直接生物认证，不显示 API Key 输入
- [ ] 使用 WebAuthn API（与 Web 端共用 Passim 的 Passkey 后端）
- [ ] App 锁: 切到后台超过 5 分钟，返回时需要重新认证
- [ ] 设置页可管理 Passkey（查看/删除/重新注册）
- [ ] 生物认证失败 → 回退到 API Key 输入
