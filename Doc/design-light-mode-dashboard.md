# Dashboard Light Mode Design Options

Dashboard 浅色模式下地球+星星背景的设计方案。

## 方案 A：天文台观测窗

左侧纯净浅色 UI，右侧深色太空区域，用渐变过渡连接。

- 左 0-50%：纯白/浅色背景，放置所有 UI 面板
- 中间 50-80%：渐变过渡带（白 → 蓝灰 → 透明），模拟观测窗边缘
- 右 80-100%：黑色太空 + 真实星星 + 3D 地球
- 卡片使用标准浅色毛玻璃

**优点**：星星真实、地球突出、UI 可读性好
**缺点**：白色和黑色太空拼接感强，难以做到自然协调
**状态**：已实现并测试，整体不协调，pass

## 方案 B：星图 / 天文图册

整体浅色底，星星反转为深灰/淡蓝小点，地球增强亮度+光晕。

- 背景：极淡蓝灰底 `oklch(0.97 0.008 255)`
- 星星：Three.js 粒子改为深色（藏蓝/灰色），size 更小，像印在纸上的天文图
- 地球：shader 提亮，外围加淡蓝 radial glow
- 可选：加极淡的轨道弧线或星座连线增加层次
- Canvas 背景改为透明，融入页面浅色底
- 卡片使用更高不透明度的浅色毛玻璃

**优点**：全局统一浅色、风格独特、精致
**缺点**：星星的闪烁感弱化，深色粒子需精细调参
**感觉**：高级天文图册、NASA 技术文档

## 方案 C：轨道日出

模拟从太空看日出的效果，上暗下亮。

- 上半部分（0-40%）：深藏蓝背景 + 密集星星 + 地球
- 中间（40-55%）：金色/暖橙日出光线过渡带
- 下半部分（55-100%）：渐变到浅色，放置 UI 面板
- 需要调整 dashboard 布局为上下结构（而非左右）

**优点**：极其戏剧性、叙事感强、自然的明暗过渡
**缺点**：布局约束大（面板只能在下方），与当前左右布局冲突
**感觉**：科幻电影、太空站控制台

## 方案 D：柔光宇宙

极淡蓝灰底 + 微弱发光粒子，地球加大范围柔和光晕。

- 背景：极淡蓝灰 + 微弱 noise 纹理增加质感
- 星星：极细微的发光粒子（CSS 或 WebGL），不追求真实
- 粒子颜色：半透明淡蓝、淡金、柔白
- 地球：3D 渲染但 Canvas 设为透明，增强亮度，加大范围柔和 halo
- 卡片：高透明度毛玻璃（比暗色模式更透）

**优点**：统一感最好、优雅温柔、最亲切
**缺点**：需精细调参，否则容易显得平淡
**感觉**：高端 SaaS、Apple 风格、云服务控制台
